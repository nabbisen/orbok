semantic recall and p99 latency require a local model artifact run and remain a
separate validation item.

## RFC-023 Decision: Exact scan for v1.0.0

At 100 documents (debug), p99 = 31 ms. Scaling to 1,000 documents in
release mode is estimated at ~20–30 ms. At 100,000 documents (extreme
case), exact scan over 384-dim FP32 vectors is ~150 ms in release mode.

**Decision: keep exact cosine scan for v1.0.0. Implement HNSW only when
benchmarks on a user's actual corpus show > 200 ms p99.**

## RFC-024 Decision: Implement INT8 quantization

At 10,000 documents × 10 chunks/doc × 1,536 bytes/chunk (384-dim FP32):
~154 MB of vector data in SQLite BLOBs. At 100,000 documents: ~1.5 GB.
INT8 reduces this 4× (~375 MB at 100k docs) with typically < 2%
recall degradation for well-trained models.

**Decision: implement INT8 as an optional storage mode for Space Saving
configuration. Default stays FP32 for accuracy.**
